# 在线 Web IDE

GLua 文档中心内置 Monaco Editor 与纯 Go WebAssembly 运行环境。所有标记为 `lua` 或 `glua` 的代码示例右上角都有“运行”按钮，点击后会把示例代码放入 Web IDE。

```lua
io.write("请输入名字：")
local name = io.read()
print("你好，" .. name)
```

## 执行与输入输出

- “执行”在独立 Web Worker 中运行代码，页面不会被脚本阻塞。
- 执行与调试会把当前 Lua/GLua 工作区快照作为内存模块系统传入 Worker；`require("lib.tools")` 可以按需加载 `lib/tools.lua`、`lib/tools.glua` 及对应目录的 `init` 文件。未被 `require` 的源码不会预编译，因此工作区中的独立测试夹具或其他 Lua 方言文件不会阻塞当前入口执行。
- 工作区快照同时作为只读虚拟文件系统提供给 `io.open`、`io.lines`、`loadfile` 和 `dofile`。例如 `io.open("./TODO.md", "r")` 可以读取资源管理器中的文件；写模式、绝对路径和包含 `..` 的路径仍会被沙箱拒绝。
- `print`、`io.write` 写入 stdout；运行错误和 `io.stderr:write` 写入 stderr。
- 运行错误会输出 Lua 原始错误对象、源码路径、行号和可用的 stack traceback，不再只显示笼统的 `lua error`。
- 脚本调用 `io.read()` 时，可以在输出框下方输入内容并发送，也可以发送 EOF。
- “停止”会销毁当前 Worker，因此死循环、等待输入以及被 `pcall` 捕获的普通中断都能强制结束。
- 调试命中断点并处于暂停状态时仍可使用“停止”，它会立即销毁 Worker 并结束本次调试。
- 可以把一个 `.lua`、`.glua` 或普通文本文件直接拖入代码编辑区域。单个文件上限为 1 MiB，目录和二进制文件会被拒绝。

## 编辑能力

- Monaco 为每个文件建立独立 Model，多标签分别保留撤销栈、光标和未保存状态。
- 标题栏提供“暗色 / 亮色”主题选择；首次打开按系统外观选择，用户手动切换后会持久保存，并同步应用到 Monaco、资源管理器、输出区和调试面板。
- 非全屏 Playground 可以拖动标题栏调整位置；窗口始终约束在浏览器可视区域内，全屏时自动禁用拖动。
- 左侧资源管理器按目录树展示文件，目录可以展开和折叠；顶部搜索框按文件名或相对路径即时过滤。资源管理器支持新建、打开、重命名和删除文件；`Ctrl+S` / `Command+S` 保存当前文件。
- 编辑器高亮规则与桌面扩展对齐，区分 Lua 5.3/GLua 关键字、标准库命名空间、函数、方法、成员属性和 GLua 事件常量。
- “格式化”复用 GLua CLI 与 LSP 的同一套 formatter；一元负号会保留为 `-1`，二元计算符号格式化为 `value - 1`；语法错误时不会覆盖原代码，错误会写入 stderr。
- `Ctrl+Space` / `Command+Space` 主动唤起补齐，输入 `.` 或 `:` 时自动提示成员或方法。
- 补齐候选会合并 WASM 语言分析结果与桌面扩展的完整 `builtin-functions.json`，包含 Lua/GLua 关键字、光标前已声明的简单文件符号、标准库和 GLua 命名空间，以及项目内置函数、常量、签名和中文说明；函数候选使用参数 snippet，支持 Tab 切换参数占位符。
- 工具栏按钮在鼠标悬停时显示功能说明和可用快捷键；没有快捷键的文件与工作区操作显示对应功能说明。
- 在 `if ... then`、循环 `do`、`function`、`repeat` 和 GLua `switch` 块头按 Enter，会按桌面扩展规则生成缩进正文与 `end`/`until`；下一有效行已有闭合语句时不会重复生成。
- WebAssembly 内的 Go 语言服务复用桌面 LSP 的 parser、codegen、内置目录与语义 token 规则，实时显示诊断、语义高亮、悬停说明和跳转定义。
- 工作区内的静态 `require("module")` 支持模块成员补齐、跳转到模块入口，以及跳转到返回表的直接成员定义；未保存的 Monaco Model 内容优先于持久化文件。

浏览器语言服务不开放网络端口，也不会额外启动 `gopls` 或 LSP 子进程。动态计算的 `require`、运行期生成字段和完整工程级引用搜索仍建议使用 VS Code 或 JetBrains 的 GLua LSP 插件。

## 工作区与本地文件

- 默认“浏览器工作区”使用 OPFS 持久保存多文件工程，刷新和关闭页面后仍可恢复。
- “打开目录”通过浏览器 File System Access API 打开用户主动选择的真实目录；只有该目录会获得读写权限。
- 页面会在 IndexedDB 中记录最后使用的工作区。浏览器工作区会直接恢复；本地目录会保存目录句柄，权限仍有效时自动恢复。
- 每个工作区分别保存打开的标签、标签顺序、活动文件、光标、选区、滚动位置和未保存草稿。磁盘内容与草稿基线不一致时保留草稿并显示冲突标记，不会自动覆盖磁盘文件。
- 本地目录权限被撤销或过期时会显示错误与“重新授权”按钮；重新授权成功后恢复目录树和该工作区的编辑会话。
- 浏览器不支持目录选择或用户取消授权时，仍可继续使用浏览器工作区和文件拖放。
- 真实目录权限由浏览器管理，重新打开页面后可能需要用户再次确认；清除站点数据会同时清除工作区记录和草稿。页面不会上传工作区文件。

## 调试

点击行号可以设置或取消断点。“调试”会立即执行代码，不在第一行自动暂停；遇到用户断点或主动点击“暂停”后提供：

- 暂停、继续。
- 单步进入、单步跳过、单步跳出。
- 当前执行行高亮。
- 当前帧局部变量查看；table 可以有限深度展开。

调试采用逐指令观察，因此会比普通“执行”模式慢。没有开启调试时不会安装指令观察器。

## 快捷键

| 快捷键 | 操作 |
| --- | --- |
| `Ctrl+Enter` / `Command+Enter` | 执行 |
| `F5` | 开始调试或继续 |
| `Shift+F5` | 停止 |
| `F10` | 单步跳过 |
| `F11` | 单步进入 |
| `Shift+F11` | 单步跳出 |
| `Alt+Shift+F` | 格式化代码 |
| `Ctrl+Space` / `Command+Space` | 唤起代码补齐 |
| `Ctrl+S` / `Command+S` | 保存当前文件 |
| `Alt+Enter` | 切换全屏 |
| `Esc` | 退出全屏 |

## 浏览器沙箱边界

Web IDE 不开放环境变量、进程执行、网络监听和 `native_modules`。OPFS 文件会持久保存；真实文件系统只限用户通过目录选择器明确授权的目录。Lua State、stdin、断点和运行期调试数据不会持久保存。
